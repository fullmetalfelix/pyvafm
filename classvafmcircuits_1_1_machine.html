Detailed Description

Virtual Machine main class.

The vitual machine is effectively a Circuit, with input/output channels, but it also contains an internal assembly of circuits that run during the Update cycle. Every machine always has the output channel 'time' created during initialisation.

Add a circuit of type 'ctype' named 'name' to the setup.

This looks into all the loaded modules whose name starts with 'vafmcircuits' and finds the first class named 'ctype'. It then instantiate the class.

  • Mandatory arguments:
    • type = string: type name of the circuit class
    • name = string: name to use for the new instance of the circuit
    • others: specific arguments depending on the particular circuit to add.
  • Optional arguments:
    • pushed = bool: defined the output behaviour model.
    • others: specific arguments depending on the particular circuit to add.
Parameters
**argkwKeyworded arguments for circuit initialisation.
Returns
Reference to the created circuit.

Example:

1 machine = Machine(name='machine', dt=0.01);
2 
3 machine.AddCircuit(type='opAdd',name='adder',factors=2, pushed=True)
4 machine.AddCircuit(type='output',name='log',file='log.log', dump=1)

def vafmcircuits.Machine.Connect (   self,
  args 
)

Connect the output of a circuit to the input of another.

The I/O channels to connect are specified with the syntax: "circuit.channel", in the *args arguments array. The first element has to be the output channel to use as source, while all the following elements refer to the destination channels.

Parameters
*argsName of the channels to connect: "circuit.channel"

Example:

1 main = Machine(name='machine', dt=0.01, pushed=True);
2 
3 main.AddCircuit(type='waver', name='osc', amp=1, freq=1)
4 main.AddCircuit(type='opAdd', name='adder', factors=2)
5 
6 main.Connect("osc.sin", "adder.in1")
7 main.Connect("osc.cos", "adder.in2")
